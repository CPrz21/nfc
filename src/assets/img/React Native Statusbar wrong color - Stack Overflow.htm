React Native Statusbar wrong color

Ask Question
up vote 0 down vote favorite

I have a react native app with multiple screens. Each component has its own statusbar to define its color (see code ex component). When starting the app the right color is shown but when I go to a screen which uses another color (black instead of white or vice versa) it changes correctly but afterwards it won't change anymore when navigating further. It seems like the color can be changed only once.

render() {
  return (
    <View style={theme.container}>
      <StatusBar
        translucent={false}
        backgroundColor={S.COLOR_STATUS_BAR_DARK}
        barStyle="dark-content"
      />
      {this.props.isFetching || this.props.isSubmitting
        ? <Spinner />
        :
        <View style={theme.container}>
          { this.renderAlarms() }
        </View>
      }
    </View>
  );
}

Am I doing something wrong here?

I think your component is not re-render.
A ---> B
When A render. Statusbar color: red
When B render. Statusbar color: blue
Back to A. A is not re-render. Statusbar color: blue,

if A component re-render, this problem will solve.

You can also change color of your Status with static setBackgroundColor function.

I will recommend to set your status color on componentDidMount or componentWillMount and componentWillUnmount to set back to previous status color as shown in following example.

componentDidMount() {
  StatusBar.setBackgroundColor('red');
}
componentWillUnmount() {
  StatusBar.setBackgroundColor('white');
}
